## 4. P0 详细 case

### C1 · 首次见 latestSha 静默写基线

- **前置**：cache 中无 entry
- **步骤**：cmdCall，fetch 返 sha=X
- **预期**：写入 latestSha=X、lastNotifiedSha=X（**三都有但相等 → 不 pending**）；stderr 无 [notice]
- **延续验证**：下次 sha 变成 Y 才提示

### C2 · Gitee SSH bug 回归

- **前置**：global 装 wind-mcp-skill from Gitee SSH。fixture `case-2-global-gitee/.agents/.skill-lock.json` (v3, sourceUrl=`git@gitee.com:...`)
- **步骤**：调任意 cmdCall，强制过 TTL
- **预期**：deriveSourceUrl 返回 SSH 直通而非误拼 `https://gitee.com/git@gitee.com/...`；fetch 走 Gitee API；stderr 升级命令 = `npx skills update wind-mcp-skill -g -y`
- **验证**：mock fetch 收到的 URL 截图；cache.latestSha 写入；exit=0

### C3 · v1 schema + project scope

- **前置**：project 装 (v1)，无 sourceUrl/installedAt，仅 computedHash
- **步骤**：cmdCall
- **预期**：deriveSourceUrl 用 source 短形式 + sourceType 拼出 sourceUrl；cache key 含 canonicalized `<cwd>/skills-lock.json`；stderr 升级命令省略 `-g`

### C5 · global+project 同 skill 双装

- **步骤**：两 lock 各一条 entry，TTL 同时到期
- **预期**：cache 有两个 key；stderr 两行升级命令（一条带 -g 一条不带）；race 下 only patch self（其他 entry 字节不变，diff 验证）

### C6 · ETag/304

- **步骤**：首次调，cache 写 etag=`W/"abc"`；第二次 TTL 到期再调，server 返 304
- **预期**：lastCheckedAt 刷新；latestSha / lastNotifiedSha 不变；不计 GitHub 限流（验 mock 收到 `If-None-Match` header）；stderr 无输出

### C8 · 多子进程并发写 race

- **步骤**：spawn 5 个 child 同时调 cmdCall
- **预期**：proper-lockfile 串行；每个 child 锁内重读、仅 patch 自己 entry；最终 cache 合法 JSON 且 5 个 lastCheckedAt 全部更新

### C9/C10 · TTL 边界

- 控 `Date.now()` 让 lastCheckedAt + 5h59m → 不 fetch；+ 6h01m → fetch。验 mock fetch 调用次数 0/1

### C15 · Lazy GC

- **前置**：cache 有 entry 指向已不存在的 lockPath
- **步骤**：任意 cmdCall
- **预期**：该 entry 静默删除；cache 文件 entry 数 -1；其他 entry 不动

## 6. 跑测试命令（node:test 原生）

```bash
# 单测（全部）
node --test test/skill-update/unit/

# 单测（单个文件）
node --test test/skill-update/unit/canonicalize-lock-path.test.js

# 集成测试
node --test test/skill-update/integration/

# 输出格式选择
node --test --test-reporter=spec test/skill-update/unit/     # 人类友好
node --test --test-reporter=tap test/skill-update/unit/      # CI 工具友好
node --test --test-reporter=junit test/skill-update/unit/    # JUnit XML

# e2e
bash test/skill-update/e2e/run-all.sh
```

> Node 18+ 内置 `node:test`，零依赖。如果未来需要 mock 库（如 nock for HTTP），用 `npm install --save-dev nock` 加。当前阶段尽量用 Node 原生 + 手写 mock。

## 9. 核心回归点（v2.3 区别于旧版的关键 case）

CI 必须每次跑：

- **C1** — 首次见 latestSha 静默写基线（修正 #3）
- **C2** — Gitee SSH URL 拼接 bug 回归（v2.3 修复点）
- **C5** — global + project 双装升级命令各自给（v2.3 设计点）
- **C8** — 并发 only-patch-self（修正 #2 race fix）

任何一条 PASS → FAIL 必须阻断 merge。
