# Via Negativa减法诊断

## 适用场景

- 当一个系统、流程、生活方式越来越复杂，而效果越来越差时
- 当你在评估健康建议、组织流程、产品功能、学习方法，怀疑"加更多"本身就是问题时
- 当你想改进某件事，但不确定该加什么时

## 核心规则

1. **先删，再加。** 在复杂系统里，新增干预常常比原问题更危险
2. **医源性损害是真实存在的。** 解决方案本身可能就是病源
3. **不需要证明每个删减都有收益，只要它在减少脆弱性就够了。**

## 操作步骤

1. **列出现有干预**：把所有额外添加的规则、功能、补剂、流程、承诺一条条写出来
2. **找副作用**：逐项问它带来了什么隐藏成本、依赖、误伤、复杂度、维护负担
3. **标记脆弱源**：凡是让系统更依赖稳定环境、更依赖单点专家、更难回滚的，优先删
4. **先做减法实验**：先移除最可疑的一项或几项，观察系统是否更轻、更稳、更清晰
5. **只保留可逆新增**：如果非加不可，就只加那些下行有限、能快速撤回的小东西

## 输出风格

不要堆改善建议。先指出最该停止什么。

可以直接说：

> 你的问题不是缺一个新方法。你的问题是身上挂了太多会反噬你的东西。先卸载。
