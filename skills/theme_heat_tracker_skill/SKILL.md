# Theme Heat Tracker Skill

## 数据依赖与执行前置条件

推荐使用万得 `wind-mcp-skill` 获取底层数据。执行前先检查本地是否已有 `wind-mcp-skill`；若没有，提示用户安装：

```bash
# GitHub
npx skills add Wind-Information-Co-Ltd/wind-skills --skill wind-mcp-skill -g -y

# Gitee 镜像（国内）
npx skills add https://gitee.com/wind_info/wind-skills.git --skill wind-mcp-skill -g -y
```


## 技能定位

本技能用于跟踪某一主题题材的热度变化，回答“这条题材现在热不热、热度来自哪里、是龙头驱动还是全面扩散、是否已经开始拥挤或退潮”。输出重点不是简单罗列相关新闻，而是帮助使用者判断题材所处阶段，以及应继续跟踪、择机参与还是降低关注。

## 执行流程

### Step 1：界定题材边界与观察周期

先明确本次要跟踪的主题是什么，并把边界定义清楚：

- 题材名称对应的核心叙事是什么
- 哪些公司属于核心受益，哪些只是边缘映射
- 是观察单一题材，还是比较多个相近题材
- 是看日内热度、周度热度，还是阶段性热度演化

若题材边界模糊，应先做核心名单与边缘名单的区分，避免把无关标的混入热度判断。

### Step 2：建立题材热度的多维观察框架

热度不应只看价格涨跌，需要从多个维度综合判断：

- 题材核心标的与跟随标的的表现强弱
- 题材内部上涨家数、持续性和梯队完整度
- 成交活跃度是否同步上升
- 新闻、政策、产业动态是否持续提供新催化
- 市场讨论热度是否集中且不断强化

这一阶段的目标是确认热度是否真实存在，而不是仅凭少数龙头表现下结论。

### Step 3：判断题材所处阶段

结合市场表现和信息流变化，判断题材当前更接近哪一阶段：

- 启动阶段：少数先行标的率先异动，市场开始形成关注
- 强化阶段：核心标的持续走强，逻辑得到重复验证
- 扩散阶段：中军、二线、边缘映射标的普遍受益
- 拥挤阶段：交易过热、分支泛化、兑现压力升高
- 退潮阶段：题材热度减弱，仅剩个别强股维持表现

阶段判断必须有清晰依据，不能只使用情绪化表述。

### Step 4：识别热度来源与延续条件

进一步说明题材之所以热，是因为什么，以及要继续热下去还需要什么条件：

- 是政策刺激、产业进展、业绩兑现还是市场风格配合
- 当前催化是一次性事件，还是可连续验证的主线
- 热度是否已经从叙事走向业绩支持
- 若后续没有新增催化，热度是否容易快速回落

### Step 5：判断拥挤度与风险释放可能

热题材常见问题不是“有没有逻辑”，而是“是不是已经太拥挤”。重点说明：

- 核心标的是否已有明显抢跑
- 边缘标的是否被过度外延
- 题材涨幅和成交是否出现脱离基本逻辑的扩张
- 一旦情绪回落，最先承压的环节会在哪里

### Step 6：形成题材热度跟踪成稿

最终输出应帮助使用者快速判断题材是值得继续跟踪、等待分歧、做高低切换，还是暂时回避。正文只保留阶段判断、驱动解释、拥挤风险与后续观察点。

## 质量要求

1. 题材热度分析必须把“价格表现、扩散层级、催化供给、拥挤程度”放在同一框架下。
2. 必须区分核心受益和边缘映射，避免题材外延无限扩大。
3. 阶段判断应基于证据，不可仅凭情绪化词语。
4. 需要同时写出延续条件和退潮风险，避免只给单边结论。
5. 输出要能服务热点跟踪和轮动判断，而不是简单热点摘要。
