# Dividend Change Explainer Skill

## 数据依赖与执行前置条件

推荐使用万得 `wind-mcp-skill` 获取底层数据。执行前先检查本地是否已有 `wind-mcp-skill`；若没有，提示用户安装：

```bash

# Gitee 镜像（国内）
npx skills add https://gitee.com/wind_info/wind-skills.git --skill wind-mcp-skill -g -y
```


## 技能定位

本技能用于解读公司分红政策或分红水平发生变化时的真实含义，重点回答“这次增加、减少、暂停或恢复分红到底说明了什么、能持续多久、对不同类型投资者意味着什么”。输出重点不是只说明派息数字，而是解释其背后的经营与资本配置逻辑。

## 执行流程

### Step 1：确认分红动作与基准状态

先确认这次变化属于哪类动作：

- 提高分红
- 削减分红
- 暂停分红
- 恢复分红
- 特别分红

同时明确变化相对的是上一次分红、过去一年平均水平，还是长期分红政策基准。

### Step 2：量化变化幅度与表面吸引力

把变化用可比较方式写清楚，重点包括：

- 每股分红变化
- 分红总额变化
- 股息率变化
- 派息率变化
- 与历史分红水平和同业的相对位置

这一阶段先回答“变了多少”，但不急于等同于利好或利空。

### Step 3：评估持续性与支付能力

真正有价值的解读要看公司是否“付得起、愿意继续付”。重点检查：

- 利润和现金流是否足以支撑当前分红
- 资产负债表、债务压力和资本开支需求是否允许
- 分红是否建立在一次性收益之上
- 管理层是否明确说明长期分红框架

### Step 4：判断分红变化背后的意图

进一步分析这次变化可能反映的经营或资本配置信号，例如：

- 现金流质量提升
- 成长机会减少，开始回馈股东
- 管理层希望稳定预期或修复信心
- 经营压力上升，被迫保留现金
- 为并购、扩产、偿债等事项腾挪资源

### Step 5：翻译成投资含义

将分红变化放到不同投资风格下理解，至少回答：

- 对收益型投资者是否更有吸引力
- 对成长型投资者是否意味着机会成本变化
- 对估值与风险偏好会产生何种影响
- 哪些情况下高股息并不代表安全

### Step 6：整合成分红解读成稿

最终输出应帮助使用者判断“这次分红变化是不是重要信号”，而不是仅停留在分红数额播报。

## 质量要求

1. 不能把“分红增加”机械等同于利好，把“分红减少”机械等同于利空。
2. 必须同时分析支付能力和管理层意图，缺一不可。
3. 若分红变化受一次性事项影响，应单独标明，避免误判持续性。
4. 对收益型与成长型投资者的含义可以不同，需明确区分。
5. 输出应回答“这次分红变化值不值得重视”，而不是只报数字。
