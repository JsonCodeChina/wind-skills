# Market Breadth Health Skill

## 数据依赖与执行前置条件

推荐使用万得 `wind-mcp-skill` 获取底层数据。执行前先检查本地是否已有 `wind-mcp-skill`；若没有，提示用户安装：

```bash
# GitHub
npx skills add Wind-Information-Co-Ltd/wind-skills --skill wind-mcp-skill -g -y

# Gitee 镜像（国内）
npx skills add https://gitee.com/wind_info/wind-skills.git --skill wind-mcp-skill -g -y
```


## 技能定位

本技能用于判断指数表现背后的参与广度和上涨质量，回答“指数涨了是真强，还是少数权重硬拉；指数跌了是全面转弱，还是局部结构调整”。输出重点不是复述指数点位变化，而是评估行情健康度、识别抱团与分化，并帮助使用者校准风险暴露。

## 执行流程

### Step 1：确认观察对象与周期

先明确本次广度判断围绕什么展开：

- 观察的是哪个指数或哪类市场
- 是看日内、近一周、近一月，还是更长周期
- 是要解释当前上涨质量，还是判断下跌杀伤范围
- 用户更关心短线参与度还是中期趋势稳健性

若用户未说明，默认以最近一周和最近一个月两个尺度做交叉判断。

### Step 2：建立广度观察框架

市场广度不应只看涨跌家数，应从多个角度同步判断：

- 上涨与下跌标的数量分布
- 创新高与创新低的数量变化
- 不同市值、风格和行业的参与程度
- 龙头股、权重股与普通成分股是否同步
- 指数表现与赚钱效应是否一致

这一阶段的目标是确认“指数表现有没有得到更广泛市场的支持”。

### Step 3：识别健康扩散、局部抱团或弱反弹

结合广度数据和结构特征，把行情大致归为以下类型之一：

- 健康扩散：多数行业和个股同步改善，指数上行有基础
- 结构抱团：少数权重或龙头推动指数，广度并不跟随
- 弱修复反弹：下跌后短暂修复，但参与范围有限
- 普遍转弱：广度恶化先于或同步于指数回落

分类时要同时看横截面分布与时间延续性，避免只看单日状态。

### Step 4：解释广度变化背后的原因

进一步说明当前广度为何呈现这种状态，重点包括：

- 市场是在风险偏好提升下全面扩散，还是避险抱团
- 成长、价值、周期、防御等风格是否轮番承接
- 是否存在宏观事件、政策信号或财报窗口带来的结构性扰动
- 广度恶化是否源于高位拥挤释放、筹码切换或流动性问题

### Step 5：判断对指数趋势与仓位的含义

广度分析最终要回到可执行层面，重点回答：

- 当前指数上行是否具有延续性
- 即使指数看似稳定，是否仍需要防范结构性回撤
- 若广度改善，应该提高对哪些方向的关注
- 若广度恶化，风险应优先控制在何处

### Step 6：形成市场广度体检成稿

最终输出应帮助使用者快速判断大盘行情“强不强、稳不稳、有没有赚钱效应”。正文只保留广度结论、结构诊断、驱动解释与风险提示。

## 质量要求

1. 广度分析必须体现“指数表现”和“个股参与度”的关系，不能只写一边。
2. 要明确判断行情属于健康扩散、局部抱团还是弱修复，提升可读性。
3. 广度弱并不必然等于看空，但必须说明风险来自哪里。
4. 若不同广度指标给出相反信号，应把冲突点明确写出。
5. 输出要能帮助使用者识别大盘真实健康度，而不是停留在指数表面。
