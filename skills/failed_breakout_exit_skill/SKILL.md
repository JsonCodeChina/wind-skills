# Failed Breakout Exit Skill

## 数据依赖与执行前置条件

推荐使用万得 `wind-mcp-skill` 获取底层数据。执行前先检查本地是否已有 `wind-mcp-skill`；若没有，提示用户安装：

```bash

# Gitee 镜像（国内）
npx skills add https://gitee.com/wind_info/wind-skills.git --skill wind-mcp-skill -g -y
```


## 技能定位

本技能用于处理突破交易中最致命也最常见的失败场景: 价格站上去了，但承接没有跟上，随后回落失守关键位。输出重点不是解释为什么看错，而是帮助使用者尽快识别假突破、收缩风险暴露，并保留再次观察的秩序。

## 功能简介

本技能的核心功能是：识别假突破并给出撤退动作。

它强调把失败信号、撤退顺序和后续重评条件写清楚，帮助使用者在突破失效时快速修正，而不是抱着“可能再拉回去”的幻想拖延。

## 使用场景

- 突破交易买入后，价格重新跌回关键位附近。
- 盘中放量冲高但尾盘回落，用户不确定该不该立刻处理。
- 次日低开失守使突破逻辑明显受损，需要快速修正。
- 团队想把假突破退出机制写成标准化纪律。

## 工作原则

1. 假突破的核心不是“跌了多少”，而是“突破前提是否被破坏”。
2. 失败撤退必须优先处理风险，再处理情绪和解释。
3. 需要区分正常回踩确认和真正的失效回落。
4. 撤退后要保留重新观察条件，避免把纠错变成情绪报复。
5. 正文只给出识别与动作框架，不讨论内部实现过程。

## 执行流程

### Step 1：确认突破原始前提

先回到交易最初的成立条件：

- 当时突破依赖的关键位、量能和环境是什么
- 当前价格回落破坏的是哪一部分前提
- 这笔交易原本属于试错突破还是确认突破
- 是否仍存在合理的回踩确认解释空间

### Step 2：识别假突破信号层级

把失败迹象分成不同严重度：

- 盘中冲高回落但收盘仍站在关键位之上
- 收盘跌回关键位下方
- 次日低开继续走弱
- 放量冲高后几乎无承接，形成明显抛压

不同层级对应的动作强度应不同。

### Step 3：区分正常回踩与失败回落

避免把所有回踩都当作失败，但也不能自我安慰：

- 回踩是否有缩量、承接和快速修复
- 跌回关键位下方后能否迅速重新站回
- 板块和环境是否仍支持再次发起
- 若量价结构明显转差，是否已不再值得等待

### Step 4：设计撤退动作顺序

一旦确认失败风险上升，应明确动作优先级：

- 是先减半还是一次性退出
- 若尾盘确认失效，是否应隔夜降低暴露
- 若次日低开进一步失守，是否直接执行剩余退出
- 是否保留极小观察仓，还是全部清空等待重建

### Step 5：补充失败后的重评条件

失败处理不等于永远拉黑，需要写清楚什么情况下还能再看：

- 重新站回关键位并得到承接
- 板块重新强化
- 出现新的催化或更优结构
- 成交结构重新改善

### Step 6：形成假突破撤退成稿

最终输出应帮助使用者知道“现在是噪音、回踩还是失败；如果是失败，该怎么退；退完以后什么时候再看”。正文保留信号分级、撤退顺序和重评条件。

## 质量要求

1. 必须区分正常回踩与失败回落。
2. 要对失败信号做层级划分，而不是只有二元判断。
3. 撤退动作必须按顺序写清。
4. 重新观察条件要具体，避免情绪性反复进出。
5. 输出应帮助快速纠错，而不是延长侥幸期。
