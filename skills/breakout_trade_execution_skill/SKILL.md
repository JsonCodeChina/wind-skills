# Breakout Trade Execution Skill

## 技能定位

本技能用于把“这只股票可能要突破了”转化成一份临盘可用的执行方案。输出重点不是泛泛讨论突破形态，而是明确突破前看什么、突破时怎么确认、突破后如何跟进，以及一旦失败应怎样快速撤退。

## 功能简介

本技能的核心功能是：为突破交易制定落地执行方案。

它强调把结构判断、量能确认、市场环境、入场节奏和失败修正连接起来，帮助使用者把追强从情绪动作变成纪律动作。

## 使用场景

- 用户准备在关键阻力位附近做突破跟随。
- 盘前需要把候选突破股写成具体执行预案。
- 盘中出现快速放量上冲，需要判断该不该跟、怎么跟。
- 团队要统一突破交易的确认口径和撤退规则。

## 工作原则

1. 突破执行必须先区分“接近突破”和“有效突破”。
2. 价格站上关键位只是开始，量能承接和环境配合同样重要。
3. 突破交易要有明确追价上限，不能无限追。
4. 失败撤退方案必须和入场方案同时写定。
5. 正文只输出执行框架，不讨论内部实现细节。

## 执行流程

### Step 1：定义突破结构与关键位

先把这笔交易的突破对象定义清楚：

- 关键阻力位或平台上沿在哪里
- 当前是箱体突破、平台突破、杯柄突破还是其他整理后的释放
- 结构是否已经足够成熟，还是仍偏早期
- 当前价格距触发位是接近、过远还是已提前透支

### Step 2：判断环境是否支持追强

突破成功率很大程度取决于环境：

- 指数与情绪是否允许高位扩散
- 板块是否存在明显的强势核心
- 市场当前偏好追随新高，还是更容易冲高回落
- 同类突破近期成功率如何

若环境不支持，应主动降低执行积极度。

### Step 3：设定触发条件与确认层级

把突破触发拆成更严谨的层级：

- 盘中触发需要满足哪些价格条件
- 是否需要同步看到放量、换手、承接等确认
- 高开直接越过关键位时，是否仍属于可执行机会
- 若突破后立刻回落，何时判定为无效

### Step 4：设计入场节奏与追价边界

突破交易最怕“追得太晚，退得太慢”。需要提前明确：

- 首笔是在刚触发时进入，还是等待回踩确认
- 允许追价的价格带和最大偏离是多少
- 是一次完成，还是必须先小仓验证
- 若首个买点错过，是否允许等待二次机会

### Step 5：制定持仓后跟进与加仓规则

一旦执行后，下一步也要提前写清：

- 哪种走势说明突破真实有效
- 哪种回踩属于健康确认，哪种回落属于危险信号
- 是否允许在确认后加仓
- 若当日尾盘未能站稳，隔夜持有是否仍合理

### Step 6：设计失败突破撤退动作

突破交易最关键的不是抓住多少成功，而是少吃失败：

- 跌回关键位下方时如何处理
- 放量冲高后无承接是否必须降仓
- 若次日低开失守，动作顺序是什么
- 是否允许保留小仓位等待重新构建

### Step 7：形成突破执行成稿

最终输出应让使用者知道“什么时候可以动、到什么程度不该追、突破后看什么、失败时怎么退”。正文保留关键位、执行条件和撤退纪律。

## 质量要求

1. 需明确区分突破前、突破中、突破后三个阶段。
2. 触发条件、追价边界和失败边界必须同时存在。
3. 环境不支持时要敢于给出不追结论。
4. 失败处理必须具体到动作顺序。
5. 输出应直接服务真实临盘执行，而不是泛化技术点评。
