# Support Break Warning Skill

## 数据依赖与执行前置条件

推荐使用万得 `wind-mcp-skill` 获取底层数据。执行前先检查本地是否已有 `wind-mcp-skill`；若没有，提示用户安装：

```bash
# GitHub
npx skills add Wind-Information-Co-Ltd/wind-skills --skill wind-mcp-skill -g -y

# Gitee 镜像（国内）
npx skills add https://gitee.com/wind_info/wind-skills.git --skill wind-mcp-skill -g -y
```


## 技能定位

本技能用于把价格接近、突破或跌破关键位置时需要关注的判断与动作整理成一份纪律化预警。重点不是简单报一个价位，而是说明这个位置为什么重要、确认标准是什么、失效后会发生什么。

## 执行流程

### Step 1：确认关键位置的定义

先明确用户所说的关键位属于哪一类：

- 前高、前低
- 平台上沿、平台下沿
- 中长期均线附近
- 趋势线、缺口、密集成交区
- 用户预设止损位、止盈位或观察位

如果用户没有给定价格，应结合近期结构先识别最有意义的位置，再进入后续判断。

### Step 2：解释位置重要性

说明该位置为何重要，至少回答以下问题：

- 它是趋势延续的确认位，还是趋势破坏的警戒位
- 附近是否可能聚集止损盘、追涨盘或解套盘
- 该位置被突破或跌破后，市场心理预期如何变化

### Step 3：判断当前接近状态

识别股票当前处于哪种状态：

- 尚未触及，但正在逼近
- 刚触及，尚未完成确认
- 已有效突破
- 已有效跌破
- 盘中穿越但收盘未确认

盘中触及与收盘确认意义不同，必须单独说明。

### Step 4：设定确认规则

为该关键位设定简单明确的确认逻辑，优先考虑：

- 收盘是否站稳或跌破
- 成交量是否放大并匹配方向
- 次日是否延续，而非冲高回落或跌破后回拉
- 板块和市场环境是否支持该方向成立

### Step 5：生成应对提示

围绕不同状态给出纪律化提示：

- 未确认前应观察什么
- 有效突破后适合如何处理
- 有效跌破后应优先控制什么风险
- 假突破或假跌破常见的识别信号是什么

### Step 6：整合为关键位预警卡

最终输出一张简洁预警卡，直接服务执行与观察，不展开技术分析过程。

## 输出结构

---

# {股票名称}关键位预警

## 一、关键位置

- **位置类型**：{支撑/压力/前高/前低/平台边界/均线}
- **关键价位**：{价格}
- **重要原因**：{为什么这个位置重要}

## 二、当前状态

- **当前位置关系**：{逼近/触及/突破/跌破/未确认}
- **量价特征**：{放量/缩量/平量，说明结构}
- **环境配合度**：{板块与市场是否支持}

## 三、确认标准

- **向上确认**：{满足什么条件才算站稳}
- **向下确认**：{满足什么条件才算失守}
- **失真信号**：{什么情况代表假突破或假跌破}

## 四、应对提示

- **当前最优先观察项**：{一句话}
- **若向上成立**：{行动提示}
- **若向下成立**：{行动提示}
- **需要回避的误判**：{常见陷阱}

---

## 质量要求

1. 关键位必须与近期结构有关，不能随意给出缺乏依据的价位。
2. 预警重点是“确认条件”，不能把盘中触碰直接写成突破成立。
3. 应对提示要体现纪律，不输出模糊表述。
4. 若结构本身不清晰，应明确说明“该位置参考意义有限”。
5. 语言应便于执行前快速阅读，避免写成长篇技术分析。
