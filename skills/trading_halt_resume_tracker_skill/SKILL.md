# Trading Halt Resume Tracker Skill

## 数据依赖与执行前置条件

推荐使用万得 `wind-mcp-skill` 获取底层数据。执行前先检查本地是否已有 `wind-mcp-skill`；若没有，提示用户安装：

```bash
# GitHub
npx skills add Wind-Information-Co-Ltd/wind-skills --skill wind-mcp-skill -g -y

# Gitee 镜像（国内）
npx skills add https://gitee.com/wind_info/wind-skills.git --skill wind-mcp-skill -g -y
```


## 技能定位

本技能用于解释一只股票为什么停牌、为什么复牌、事件进展到了哪一步，以及复牌后最需要关注什么。目标是把停复牌从“事件名称”转化为“对股价和预期的实际影响框架”。

## 执行流程

### Step 1：确认停复牌状态

先明确事件属于哪一种：

- 停牌
- 临时停牌
- 长时间停牌
- 复牌
- 复牌前最新进展披露

同时确认事件发生时间、持续时间以及当前是否仍处于信息不完整阶段。

### Step 2：识别触发原因

重点梳理触发停复牌的核心原因：

- 重大资产重组、并购、定增、控制权变化
- 重大公告待披露
- 股价异常波动、监管核查
- 财务或治理风险
- 退市风险、债务风险、重大诉讼

若原因为多重因素叠加，要识别主因，不要平均展开。

### Step 3：梳理事件时间线

将事件按时间线重构，说明：

- 何时首次停牌或临停
- 期间披露了哪些关键信息
- 当前信息透明度到了什么程度
- 复牌是否意味着核心不确定性已部分消除

### Step 4：评估复牌影响机制

停复牌事件的核心不在“发生了什么”，而在“市场会如何定价”。需要重点判断：

- 是利好兑现、风险释放，还是不确定性延后
- 复牌后更可能面临补涨、补跌、震荡换手还是持续博弈
- 是否存在流动性冲击、情绪宣泄或监管约束

### Step 5：提炼复牌后观察框架

若已复牌或即将复牌，明确给出：

- 开盘最需要关注的价格和成交信号
- 事件主线是否足够强到支持持续交易
- 哪些表现说明市场仍不认可当前披露内容

### Step 6：整理成停复牌跟踪摘要

最终输出应体现“原因 - 进展 - 影响 - 后续观察”的闭环，不展示信息搜集过程。

## 输出结构

---

# {股票名称}停复牌跟踪

## 一、事件状态

- **当前状态**：{停牌/临停/复牌/待复牌}
- **触发时间**：{日期时间}
- **核心原因**：{一句话}

## 二、事件时间线

| 时间 | 进展 | 对市场理解的影响 |
| :--- | :--- | :--- |
| {时间} | {事件} | {影响} |

## 三、影响判断

- **事件性质**：{偏利多/偏利空/中性/仍不确定}
- **不确定性是否下降**：{高/中/低}
- **复牌后主要风险**：{风险}
- **复牌后主要机会**：{机会}

## 四、复牌后观察框架

- **最先看什么**：{价格/成交/承接/回补}
- **走强确认信号**：{信号}
- **转弱或失败信号**：{信号}

---

## 质量要求

1. 不要只停留在事件名称层面，要解释其对定价的实际影响路径。
2. 若披露仍不充分，要明确写出剩余不确定性在哪里。
3. 复牌判断要特别关注流动性和情绪冲击，不可只看公告措辞。
4. 长时间停牌与短时临停的解读框架要区分。
5. 输出要帮助用户构建后续观察，而不是只完成新闻摘要。
