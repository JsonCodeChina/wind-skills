# After Close Watchlist Recap Skill

## 数据依赖与执行前置条件

推荐使用万得 `wind-mcp-skill` 获取底层数据。执行前先检查本地是否已有 `wind-mcp-skill`；若没有，提示用户安装：

```bash
# GitHub
npx skills add Wind-Information-Co-Ltd/wind-skills --skill wind-mcp-skill -g -y

# Gitee 镜像（国内）
npx skills add https://gitee.com/wind_info/wind-skills.git --skill wind-mcp-skill -g -y
```


## 技能定位

本技能用于把一组自选股的收盘后复盘工作标准化，帮助使用者回答三件事：今天谁强谁弱、强弱背后因为什么、明天还值不值得继续看。目标是为下一交易日输入更清晰的观察清单，而不是只做涨跌幅汇总。

## 执行流程

### Step 1：回顾当日表现

逐只检查当日的关键表现，包括：

- 涨跌幅、振幅、成交变化、换手变化
- 日内是否出现冲高回落、探底回升、尾盘异动
- 与所属板块、指数相比是领涨、跟涨还是明显掉队

### Step 2：解释表现驱动

优先判断今日表现主要由哪类因素驱动：

- 新增消息或公告
- 板块轮动或市场风格切换
- 技术位置触发
- 资金博弈或情绪宣泄
- 无明显催化，仅随市场波动

### Step 3：识别强弱性质

把强弱分成更有决策意义的类型，而不只是看涨跌：

- 主动走强：脱离板块仍强
- 被动跟涨：板块带动明显
- 承压转弱：反弹无力或跌破结构
- 噪音波动：无明确主线变化

### Step 4：提炼次日观察点

针对每只值得继续跟踪的股票，提炼一个最关键的次日验证点，例如：

- 是否能延续放量强势
- 是否会回补缺口
- 是否守住新形成的关键位
- 是否还有事件催化延续

### Step 5：更新关注优先级

给出次日关注等级调整建议：

- 提高优先级：今天出现新主线、新强度或新催化
- 维持优先级：逻辑未变，但暂无增量信号
- 降低优先级：结构破坏、催化落空或关注度明显衰退

### Step 6：输出收盘复盘摘要

最终输出应以“结果 + 解释 + 次日观察”为主，不展示逐步排查过程。

## 输出结构

# 自选股收盘复盘（{YYYY-MM-DD}）

- **最强观察对象**：{股票/原因}
- **最弱观察对象**：{股票/原因}
- **最值得带到明天的线索**：{线索}

## 二、分股票复盘

| :--- | :--- | :--- | :--- | :--- | :--- |

- {市场风格是否变化}

## 四、明日准备

- **明天优先盯盘名单**：{股票列表}

## 质量要求

1. 复盘必须回答“为什么今天会这样”，而不只是记录结果。
2. 结论要服务次日准备，避免写成纯记录型日报。
3. 没有增量变化的股票可简写，不要平均用力。
4. 强弱判断要结合板块与市场环境，不能脱离背景孤立评价。
5. 若找不到明确驱动，应写明“暂无明确新增驱动”，避免过度解释。
