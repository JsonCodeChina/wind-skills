# Price Target Reach Alert Skill

## 数据依赖与执行前置条件

推荐使用万得 `wind-mcp-skill` 获取底层数据。执行前先检查本地是否已有 `wind-mcp-skill`；若没有，提示用户安装：

```bash
# GitHub
npx skills add Wind-Information-Co-Ltd/wind-skills --skill wind-mcp-skill -g -y

# Gitee 镜像（国内）
npx skills add https://gitee.com/wind_info/wind-skills.git --skill wind-mcp-skill -g -y
```


## 技能定位

本技能用于在股价接近、达到或超过目标价时，把“要不要卖、卖多少、要不要重新看目标”这类问题整理成有纪律的处理框架。重点不是机械地到价即卖，而是判断目标价触发后当前赔率、预期兑现度和持仓处理优先级。

## 执行流程

### Step 1：确认目标价背景

先明确目标价从何而来，以及它代表什么：

- 研究目标价
- 交易计划中的止盈位
- 技术目标位
- 用户自设观察位

若目标价来源不清，先说明其参考意义有限，再进入处理建议。

### Step 2：判断当前触发状态

识别股价与目标价的关系：

- 尚未触达，但已接近
- 刚好触达
- 短暂穿越
- 明显超过
- 触达后回落

不同触发状态对应的处理优先级不同。

### Step 3：评估目标是否已被兑现

围绕“市场是不是已经交易完这段预期”来判断：

- 原先驱动股价上涨的核心逻辑是否已充分反映
- 当前价格是否仍有新增催化支撑
- 达到目标价时量价表现是强势扩张还是冲高衰竭
- 板块环境是否仍支持继续上修预期

### Step 4：给出分批处理框架

根据触发质量给出处理建议的方向：

- 若只是接近目标价：以观察和验证为主
- 若目标价被平稳触达：考虑分批兑现与保留跟踪仓位
- 若强势穿越目标价：考虑重新评估目标，而不是机械一次性处理
- 若触价即回落：重点提示预期兑现和回撤风险

### Step 5：补充再评估条件

若判断需要重估目标价，应说明哪些条件满足时才合理上修，例如：

- 基本面验证超预期
- 新增催化显著强于原假设
- 结构突破并得到持续承接

### Step 6：输出目标价触发卡

最终输出应明确“当前状态、处理建议、保留条件、失效条件”，不展示内部推导过程。

## 输出结构

---

# {股票名称}目标价触发提示

## 一、触发状态

- **目标价**：{价格}
- **当前关系**：{接近/触达/穿越/回落}
- **目标价性质**：{研究目标/交易目标/技术目标/自设目标}

## 二、当前判断

- **预期兑现度**：{高/中/低}
- **继续上行空间判断**：{高/中/低}
- **回撤风险判断**：{高/中/低}

## 三、处理建议

- **建议动作**：{观察/分批兑现/保留仓位/重新评估}
- **动作理由**：{一句话说明}
- **若继续持有，重点看**：{验证信号}
- **若开始兑现，优先原则**：{纪律提示}

## 四、再评估条件

- {什么条件下可以重新抬高预期}
- {什么条件下应确认目标价已基本完成}

---

## 质量要求

1. 到价处理必须同时考虑预期兑现度和新增催化，不能机械处理。
2. 分批建议应体现纪律性，避免模糊表述。
3. 若目标价本身依据薄弱，要明确提示其参考价值有限。
4. 对“强势穿越目标价”和“触价即回落”要给出不同框架。
5. 输出应服务执行，不写成泛泛估值评论。
