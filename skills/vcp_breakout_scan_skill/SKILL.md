# VCP Breakout Scan Skill

## 数据依赖与执行前置条件

推荐使用万得 `wind-mcp-skill` 获取底层数据。执行前先检查本地是否已有 `wind-mcp-skill`；若没有，提示用户安装：

```bash
# Gitee 镜像（国内）

```


## 技能定位

本技能用于寻找波动逐步收缩、供给逐步减弱、价格结构趋于成熟的突破预备股。输出重点不是泛泛寻找整理形态，而是识别哪些股票已经进入“压缩波动、等待释放”的关键阶段，以及这种结构是否具备较好的后续突破条件。

## 功能简介

本技能的核心功能是：筛选波动收敛充分、具备突破条件的 VCP 预备股。

它强调把波动压缩、回调幅度递减、量能收敛、供需改善和市场环境整合起来，帮助使用者建立一组高质量的突破预备股名单。

## 使用场景

- 用户采用波动收缩策略寻找主升前的准备期股票。
- 强势股回调后，需要判断哪些已经完成充分洗盘。
- 团队要筛选“离突破只差一步”的技术结构候选。
- 需要将图形观察转化为纪律化的观察清单。

## 工作原则

1. VCP 结构的核心是波动和供给收敛，不能只看价格横盘。
2. 必须检查回调幅度是否逐步收窄，而不是一次大跌后的静止。
3. 量能变化与结构位置要同步判断，不能把成交极低简单理解为健康。
4. 需要结合基本面或催化背景确认其不是纯技术噪音。
5. 正文中不显式提及内部实现方式。

## 执行流程

### Step 1：确认筛选对象与时间周期

先明确扫描目标是：

- 全市场 VCP 预备股
- 某个板块或风格中的结构筛选
- 前期强势股的二次整理
- 还是自选股里的重点技术结构复查

同时确认用户关注的是日线级别还是更高周期的收缩结构。

### Step 2：识别前置上涨基础

VCP 结构通常出现在已有上涨基础的股票上，因此要先确认：

- 股票在进入收缩期前是否已经存在清晰上涨段
- 前期上涨是否具备基本面、题材或资金主线支持
- 是否属于真正的强势股整理，而非长期弱势横盘

没有前置强势基础的横盘，很难构成高质量 VCP 候选。

### Step 3：检查波动是否逐级收缩

这是本技能最核心的判断环节。要重点观察：

- 每一轮回调幅度是否较上一轮更浅
- 每次修复后的高点是否更接近前高
- 日内和区间波动是否呈现压缩趋势
- 整理后期是否明显减少剧烈震荡

真正的波动收缩应体现“卖压在减少”，而不是“交投停滞”。

### Step 4：检查量能是否同步收敛

波动收缩若没有量能配合，可信度会明显下降。应判断：

- 回调阶段的量能是否逐步萎缩
- 反弹阶段是否偶有适度放量显示承接
- 临近关键位时是否出现量能温和回升
- 是否存在某一次放量下跌破坏供需平衡

### Step 5：定义关键位置与触发条件

对每只候选，需要明确：

- 真正的突破触发位在哪里
- 哪个价格区域属于最关键的供需转折点
- 突破时需要出现什么量价确认
- 如果未能有效站稳，应如何定义失败

### Step 6：补充基本面或催化支撑

高质量 VCP 候选，往往不只是技术结构漂亮。还要确认：

- 是否存在业绩、产品、题材、订单或行业景气支撑
- 板块是否处于强化周期
- 若结构成立，市场是否有理由给予持续关注

### Step 7：按成熟度分层排序

将候选分为：

- 结构成熟、接近触发的重点对象
- 收缩正在形成、仍需时间的中间对象
- 结构看似相似但供需不够健康的谨慎对象

### Step 8：整合成波缩突破成稿

最终输出应帮助使用者知道“哪几只票的波动收缩最有质量、等什么、怎么确认、何时放弃”。正文只保留判断与观察框架。

## 质量要求

1. 必须同时评估波动压缩与量能收敛。
2. 不能把普通横盘或弱势震荡误判为 VCP。
3. 需明确关键位置、确认条件和失效边界。
4. 应补充基本面或题材支撑，避免纯图形化结论。
5. 输出要服务突破预备股的连续跟踪。
