# Intraday Abnormal Move Alert Skill

## 数据依赖与执行前置条件

推荐使用万得 `wind-mcp-skill` 获取底层数据。执行前先检查本地是否已有 `wind-mcp-skill`；若没有，提示用户安装：

```bash
# GitHub
npx skills add Wind-Information-Co-Ltd/wind-skills --skill wind-mcp-skill -g -y

# Gitee 镜像（国内）
npx skills add https://gitee.com/wind_info/wind-skills.git --skill wind-mcp-skill -g -y
```


## 技能定位

本技能用于在盘中出现快速上涨、快速下跌、放量冲击或换手异常时，给出一份“先解释、再分级、后提示动作”的快判结果。核心目标是帮助使用者判断这波异动是噪音、事件驱动，还是值得持续跟踪的有效变化。

## 执行流程

### Step 1：定义异动对象与异动形态

先明确是哪只股票、哪一个时间段、哪一种异动。异动形态至少要描述清楚以下信息：

- 价格方向：急拉、急跌、冲高回落、探底回升、震荡放大
- 幅度与速度：短时波动是否显著偏离常态
- 成交变化：是否同步放量、缩量、换手抬升
- 所处位置：发生在前高、前低、均线密集区还是趋势中继区

### Step 2：排查显性驱动

优先检查是否存在能直接解释异动的新增信息，重点覆盖：

- 公司公告、行业新闻、政策消息、监管动态
- 财报、业绩预告、产品发布、重大合同、停复牌信息
- 板块集体异动、指数或期货联动、海外映射

如果找不到明显催化，也要明确说明“暂未发现新增显性驱动”，不能假设一定有消息。

### Step 3：判断异动是否有板块或情绪共振

分析该股票的异动是孤立事件还是群体行为，重点回答：

- 同板块是否同步异动
- 题材热度是否正在扩散
- 是否与市场主线方向一致
- 是否只是资金在局部抱团或高位切换

孤立异动与板块共振的意义完全不同，必须区分。

### Step 4：评估持续性与可信度

从“事件强度、量价结构、位置关系、跟风程度”四个角度评估异动质量：

- 事件是否足够新、足够强、足够具体
- 放量是否匹配价格突破，而非尾盘或瞬时脉冲
- 当前价格位置是否容易遭遇套牢盘或获利盘
- 市场是否给予持续跟单，而非一笔单拉抬

### Step 5：给出应对提示

输出的重点不是直接下结论买卖，而是告诉用户此时应优先观察什么：

- 若异动可信，应继续盯哪些确认信号
- 若异动质量一般，应防范哪些回撤或反抽陷阱
- 若异动高度存疑，应如何降低关注优先级

### Step 6：整合成盘中快判卡片

最终输出一份简洁结论卡，不展开内部排查过程。语言应短促、明确、便于盘中快速阅读。

## 输出结构

---

# {股票名称}盘中异动快判（{HH:mm}）

## 一、异动概况

- **异动类型**：{急拉/急跌/冲高回落/探底回升}
- **异动描述**：{在多长时间内出现了怎样的价格和成交变化}
- **所处位置**：{关键位、区间中部、前高附近、前低附近等}

## 二、可能驱动

- **显性驱动**：{若有，写明核心事件；若无，写“暂未发现新增显性驱动”}
- **板块共振**：{是否有，表现如何}
- **情绪因素**：{是否属于题材发酵、资金抢筹、恐慌踩踏等}

## 三、异动质量判断

| 维度 | 判断 | 说明 |
| :--- | :--- | :--- |
| 事件强度 | 高/中/低 | {原因} |
| 量价匹配 | 高/中/低 | {原因} |
| 持续概率 | 高/中/低 | {原因} |
| 回落风险 | 高/中/低 | {原因} |

## 四、盘中应对重点

- **优先确认信号**：{继续走强需要看到什么}
- **失效信号**：{若出现什么，说明异动大概率失真}
- **当前结论**：{一句话总结是否值得持续跟踪}

---

## 质量要求

1. 快判必须优先回答“为什么动”和“这波能不能信”，而不是重复分时表现。
2. 找不到原因时要如实说明，不允许把猜测包装成事实。
3. 对持续性的判断必须带依据，不能只写“看起来很强”。
4. 正文应适配盘中阅读，避免长段背景介绍。
5. 若是整个板块共振引发的异动，要突出“个股只是映射”还是“个股具备独立催化”。
