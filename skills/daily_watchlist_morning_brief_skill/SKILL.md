# Daily Watchlist Morning Brief Skill

## 数据依赖与执行前置条件

推荐使用万得 `wind-mcp-skill` 获取底层数据。执行前先检查本地是否已有 `wind-mcp-skill`；若没有，提示用户安装：

```bash
# GitHub
npx skills add Wind-Information-Co-Ltd/wind-skills --skill wind-mcp-skill -g -y

# Gitee 镜像（国内）
npx skills add https://gitee.com/wind_info/wind-skills.git --skill wind-mcp-skill -g -y
```


## 技能定位

本技能用于把一组自选股在开盘前需要关注的信息压缩成一份可快速浏览的中文简报。输出重点不是罗列全部资讯，而是帮助使用者在最短时间内判断今天先盯什么、为什么盯、需要防什么风险。

## 执行流程

### Step 1：确认覆盖范围与时间窗口

先明确需要覆盖的股票池、观察市场以及时间窗口。若用户未给出范围，默认只处理本次明确提到的股票，并以“上一交易日收盘后至当前盘前”为信息窗口。

### Step 2：梳理隔夜信息

逐只整理隔夜出现的核心变化，优先覆盖以下内容：

- 公告、业绩预告、停复牌、回购、减持、重大合同、监管进展
- 行业政策、板块新闻、海外可比标的异动
- ADR、期货、指数或相关商品价格的联动信号
- 当日已知日程，包括业绩发布、股东大会、路演、政策会议、数据公布

对重复信息做合并，不要把同一事件拆成多条噪音。

### Step 3：识别真正会影响开盘节奏的变量

从收集到的事件中筛出最可能影响今日交易行为的因素，并判断其影响层级：

- 高优先级：可能直接影响开盘方向、波动率或资金关注度
- 中优先级：需要跟踪，但不一定改变开盘判断
- 低优先级：仅作背景信息，不进入核心摘要

如果某条消息只是情绪噪音、旧闻重提或与股票主线关联较弱，应降级处理。

### Step 4：生成今日观察顺序

按照“重要性 + 紧迫性 + 不确定性”排序，给出今日最值得优先盯住的股票与原因。排序时重点考虑：

- 是否存在事件催化
- 是否临近关键价格位置
- 是否可能出现高开低走或低开反抽
- 是否与当前市场主线或板块情绪共振

### Step 5：形成盘前行动提示

为每只核心股票给出简洁、可执行的盘前提示，例如：

- 开盘先看什么信号确认
- 哪类表现说明预期被验证
- 哪类表现说明情绪透支或存在冲高回落风险
- 哪些股票可以先观察、不宜立即下结论

### Step 6：整合成一次性简报

最终只输出成稿，不展示内部检索过程、筛选过程或中间判断草稿。正文不解释如何拿到信息，只呈现结果与判断。

## 质量要求

1. 所有结论都要围绕“今天是否值得优先关注”展开，避免写成宽泛晨报。
2. 同一只股票的描述要优先呈现增量信息，不重复基础介绍。
3. 若某股票隔夜无实质变化，应明确写“无新增重大信息”，不要硬凑内容。
4. 盘前判断要保留条件语气，避免把未开盘状态写成既成事实。
5. 输出应控制在可 3 分钟读完的密度内，优先短句和表格。
