# Earnings Preview Skill

## 数据依赖与执行前置条件

推荐使用万得 `wind-mcp-skill` 获取底层数据。执行前先检查本地是否已有 `wind-mcp-skill`；若没有，提示用户安装：

```bash
# GitHub
npx skills add Wind-Information-Co-Ltd/wind-skills --skill wind-mcp-skill -g -y

# Gitee 镜像（国内）
npx skills add https://gitee.com/wind_info/wind-skills.git --skill wind-mcp-skill -g -y
```


## 技能定位

本技能用于在财报发布前，把“市场在等什么、公司最该证明什么、如果结果不同会怎样”整理成一份高密度前瞻底稿。输出重点不是泛泛罗列财务指标，而是帮助使用者在财报前明确预期基线、关键验证点、潜在预期差和会前应重点关注的问题。

## 执行流程

### Step 1：确认分析对象、财报周期与使用目的

先确认股票名称、上市地、本次对应的财报周期，以及用户此时要解决的问题：

- 是为了会前快速准备
- 是为了判断是否值得参与财报交易
- 是为了给团队做前瞻讨论底稿
- 是为了在持仓前明确风险暴露

若用户未说明，默认按“财报公布前的研究准备”展开，并聚焦最近一次待发布财报。

### Step 2：建立市场预期基线

先把市场已经形成的预期整理清楚，重点包括：

- 收入、利润、利润率等核心指标的市场预期区间
- 管理层上次给出的指引或暗示
- 卖方与市场讨论最集中的关键经营变量
- 上一季度财报后遗留的验证任务
- 当前股价隐含的预期温度是保守、中性还是偏热

这一阶段的目标不是直接下结论，而是先明确“市场已经知道什么、已经押注什么、还在争论什么”。

### Step 3：拆解本次财报最重要的验证点

围绕公司当前阶段，提炼 3-5 个真正决定财报解读方向的核心问题。优先关注：

- 本季度业绩是否只是表面超预期，还是质量也同步改善
- 增长来源是量、价、结构还是一次性因素
- 利润率、费用率、现金流是否支持当前增长叙事
- 订单、用户、产能、库存、开店、出货等经营变量是否验证主线
- 管理层是否需要在本次财报中回应市场最大的疑虑

验证点必须足够具体，能够在财报正文、业绩会或后续公告里被验证，而不是宽泛口号。

### Step 4：做情景推演与反应框架

在预期基线和关键变量明确后，对财报结果做条件化推演，至少覆盖：

- 明显超预期时，真正的惊喜会来自哪里
- 略超预期但股价不涨时，市场可能在担心什么
- 表面达标但指引偏弱时，解读应如何调整
- 若核心经营变量不及预期，市场会优先交易哪条风险链条
- 哪些情况下需要把“好财报”和“好股票表现”区分开看

这一步的重点是把“结果”与“市场反应”拆开处理，避免把财报前瞻写成单向押注。

### Step 5：形成会前关注清单

基于前述分析，整理出财报前最值得带着去看的问题，包括：

- 财报发布后要先核对的三个数字
- 业绩会最值得听的两个问题方向
- 哪些措辞变化会明显改变判断
- 哪些风险即使财报不错也不能忽略
- 若结果模糊，应等待哪些补充信息再下结论

### Step 6：整合成前瞻成稿

最终输出应是面向行动的前瞻底稿，不展示中间检索路径、信息收集过程或内部草稿。正文不需要解释信息如何获得，只需要呈现预期、分歧、推演和行动重点。

## 质量要求

1. 前瞻必须围绕“预期差”展开，而不是重复公司背景资料。
2. 关键看点应尽量具体到指标、业务模块或管理层表态方向。
3. 要区分“财报结果好坏”和“股价反应强弱”这两个层次。
4. 若市场预期并不清晰，应明确写出预期不一致，而不是强行给出单一口径。
5. 文本要能直接服务财报前准备与会后验证，避免写成泛泛研究摘要。
