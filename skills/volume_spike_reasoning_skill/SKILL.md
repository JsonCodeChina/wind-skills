# Volume Spike Reasoning Skill

```bash
# Gitee 镜像（国内）

```


## 技能定位

本技能用于解释“为什么突然放量”，并把放量背后的性质区分为事件驱动、板块扩散、主力博弈、技术位释放或情绪交易。目标是避免把所有放量都当成利好，也避免错过真正有信息含量的放量。

## 执行流程

### Step 1：确认放量发生的上下文

先定义放量发生的时间粒度与市场位置：

- 盘中瞬时放量、半小时放量、全天放量
- 放量发生在上涨、下跌、横盘还是突破/跌破附近
- 当日是否叠加高换手、跳空、封板、炸板或尾盘异动

### Step 2：判断放量是否显著偏离常态

比较当前成交表现与近阶段常态水平，重点看：

- 成交额或成交量是否显著抬升
- 换手率变化是否同步明显
- 放量是持续型还是脉冲型
- 放量是否主要集中在单一时段

### Step 3：排查增量驱动来源

重点从四类原因进行排查：

- 事件驱动：公告、新闻、政策、财报、产品、订单、监管
- 板块驱动：同行集体异动、题材发酵、龙头带动
- 技术驱动：关键位突破、跌破、止损踩踏、趋势加速
- 资金驱动：情绪抢筹、游资接力、博弈换手、尾盘资金试盘

### Step 4：识别放量质量

放量本身不代表有效，必须回答：

- 价格是否顺着放量方向推进
- 是否出现“只放量不走价”或“放量冲高回落”
- 放量后是否带来结构重估，而不是一天脉冲
- 当前市场是否支持这种放量延续

### Step 5：输出跟踪重点

给出后续最重要的验证问题，例如：

- 次日是否承接
- 关键位置是否站稳
- 是否继续获得题材热度支持
- 成交是否回落到常态后仍能维持价格强势

### Step 6：整合为归因结果

最终输出以“放量性质 + 可信度 + 后续观察点”为主，不展示内部排查路径。

## 质量要求

1. 归因必须覆盖多种可能，不允许一上来就把放量等同于利好。
2. 若证据不足，允许给出“归因暂不充分”的判断。
3. 解释重点是“放量为何发生”以及“是否值得继续跟踪”。
4. 同时存在多重原因时，要明确主因与次因，不要并列堆砌。
5. 输出应服务盘中快速判断，结论要短而稳。
