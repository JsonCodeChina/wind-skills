# Industry Chain Signal Skill

## 数据依赖与执行前置条件

推荐使用万得 `wind-mcp-skill` 获取底层数据。执行前先检查本地是否已有 `wind-mcp-skill`；若没有，提示用户安装：

```bash
# GitHub
npx skills add Wind-Information-Co-Ltd/wind-skills --skill wind-mcp-skill -g -y

# Gitee 镜像（国内）
npx skills add https://gitee.com/wind_info/wind-skills.git --skill wind-mcp-skill -g -y
```


## 技能定位

本技能用于把行业链条拆成上游、中游、下游或关键环节，识别景气变化如何传导，并回答“价值现在落在哪一段、下一步会传到哪里、哪些公司受益、哪些公司有风险”。输出重点不是写一份宽泛行业介绍，而是把传导路径、关键变量和错配机会说清楚。

## 执行流程

### Step 1：确认产业链范围与研究起点

先明确本次研究覆盖的产业链范围：

- 研究的是哪条产业链
- 用户更关注哪个环节或哪个公司
- 是要寻找机会、验证景气，还是识别风险
- 观察周期是短期价格变化，还是中期景气趋势

若用户只给出一家公司，也应先把它放回所在产业链中再分析，避免孤立判断。

### Step 2：搭建产业链结构图

把产业链拆成关键环节，并说明每一环节的功能与价值获取方式，通常包括：

- 上游资源、原材料或核心零部件
- 中游制造、集成、平台或分销环节
- 下游终端需求、客户结构和应用场景
- 链条中真正决定利润弹性的关键瓶颈环节

这一阶段的目标是先明确“谁给谁供给、谁对谁定价、谁最容易受影响”。

### Step 3：提取各环节的核心信号

围绕景气与利润传导，优先观察以下信号：

- 价格变化是否反映供需紧张或缓和
- 订单、出货、排产、开工率等经营线索是否同步改善
- 库存变化是主动补库、被动累库还是去库存末端
- 毛利率或利润率改善是来自量增、价升还是成本下降
- 下游需求变化是否已经向上游或中游反馈

各环节信号应尽量对应到可验证变量，避免停留在抽象判断。

### Step 4：分析传导路径与节奏错位

产业链机会常来自传导的先后顺序。要重点说明：

- 当前景气变化最先出现在哪一环
- 哪些环节已经兑现，哪些环节尚未被市场充分定价
- 传导为何可能中断、延迟或反向发生
- 哪些环节会出现“收入先改善、利润后改善”或相反的错位

### Step 5：提炼链条中的受益方、承压方与关键验证点

在传导路径明确后，提炼：

- 当前最直接受益的环节与代表公司
- 可能受压制的环节与风险来源
- 若判断成立，接下来最值得观察的验证变量
- 若判断失效，最先暴露问题的链条位置

### Step 6：形成产业链信号成稿

最终输出应帮助使用者把链条看清楚，知道应盯哪一环、信哪一类信号、回避哪类表面繁荣。成稿只呈现链条结构、关键变量、传导结论和观察框架。

## 质量要求

1. 产业链分析必须体现“结构、信号、传导、验证”四个层次，不能只写行业概览。
2. 要明确价值落点和传导方向，避免把所有环节平均处理。
3. 需区分已经兑现的景气与尚未兑现的预期，提升判断可操作性。
4. 若链条信号彼此冲突，应把冲突点写清楚，而不是强行统一。
5. 输出要能直接服务链条研究、比较环节机会和验证景气变化。
